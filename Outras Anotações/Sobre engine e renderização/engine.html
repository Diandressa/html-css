Browser engine

Dentro de cada navegador (Chrome, Firefox, Opera, etc.) existe uma implementação do motor (engine). Ele é o responsável por descobrir quais são e como renderizar as informações dos arquivos que ele recebeu para que você visualize ou interaja com os mesmos.
Exemplos de motores:

  • Gecko
  • Blink
  • Webkit

Para entender

No entanto, dentro de um navegador, há um software que descobre o que exibir para você com base nos arquivos que recebe. Isso é chamado de mecanismo do navegador.
O mecanismo do navegador é um componente de software central de todos os principais navegadores, e diferentes fabricantes de navegadores chamam seus mecanismos por nomes diferentes. O mecanismo do navegador para Firefox é chamado Gecko, e o do Chrome é chamado Blink, que por acaso é um fork do WebKit.
